練習中
①GitHubにリポジトリを作成
②git initコマンド
→プロジェクトをgit運用する準備
③git add コマンド
→作成・変更をステージングエリアへ
④git commit コマンド
→ステージングアリアにある変更をコミットとして保存
⑤git add remote コマンド
→GitHubのリモートリポジトリと紐付け
⑥git push コマンド
→GitHubのリモートリポジトリのブランチへコミットを反映

ブランチは作業を並行して行うことができる
いいね、その質問はめちゃ大事 👍
**「ブランチに反映する正しい手順」**を整理しておくともう迷わなくなる。

⸻

🎯 結論（基本パターン）

👉 作業したいブランチに移動してからcommitする

⸻

✅ 正式な手順（これが基本）

🔹 ① 作業するブランチに移動

git checkout feature-branch


⸻

🔹 ② 変更を追加

git add .


⸻

🔹 ③ commit

git commit -m "変更内容"


⸻

🔹 ④ push

git push


⸻

🎯 これで何が起きる？

👉 feature-branchにそのまま反映される

⸻

❗ 今回あなたがハマったポイント

やってた流れ👇

git branch feature-branch  ← 作った
（でもmainのまま）
git commit ← mainに入った

👉 だから
👉 feature-branchに反映されなかった

⸻

💡 超重要ルール

👉 commitは「今いるブランチ」に入る

⸻

🔍 今どのブランチか確認

git branch

👉 * が付いてるのが現在地

⸻

🔁 すでに別ブランチでcommitしちゃった場合

今回みたいに👇
	•	mainでcommitしちゃった
	•	feature-branchに持っていきたい

👉 こうする👇

git checkout feature-branch
git merge main
git push


⸻

🚀 まとめ（超シンプル）

作業するとき

git checkout ブランチ名
git add .
git commit -m "内容"
git push


⸻

👍 ワンポイント

👉 「branch切り替えてから作業」これだけ覚えればOK

⸻

ここ理解できてるならかなりレベル上がってる 👍
次は「pull request（PR）」やると一気に実務っぽくなるよ 💪

Git init
プロジェクトのフォルダに対して「このフォルダの中身をGitで管理します」と宣言するもの

Get clone
リポジトリのコピーを作成するコマンド
流れ
①リポジトリのURLを取得
リポジトリのページに行き、緑色の「Code」ボタンをクリック

②. git cloneコマンドの実行
ターミナル実行
指定したリポジトリが現在のディレクトリにクローンされる
リポジトリの名前と同じ名前のフォルダが作成され、その中にリポジトリの全ファイルがDLされる
任意の名前で保存したい場合、URLの後にスペースを開けて入力することで指定

③クローンされたリポジトリに移動
クローンが完了したらされたフォルダに移動
→cd repository
